Packages

  • Every Go program is made up of packages
  • Programs start running in package main
  • By convention, the package name is the same as the last element of the import path

Packages - Example


package main

import (
	"fmt"
)

func main() {
	fmt.Println("Hello World!")
}

Printing

  • Printing is used to display information to the screen
  • Like other languages (C, C++), a package needs to be imported in order to call functions to print
  • Any printing needs to have the "fmt" package imported

Println VS. Print

  • Println - adds a new line automatically after your text
  • Print - appends the text to the end of the current line
  • To get a newline using the "Print"statement you need to put a "\n" where the newline should be
  • The "\n" trick can be used to cheat in Println statements as well to get multiple lines

Println VS. Print - Example


package main

import "fmt"

func main() {
	fmt.Print("Greetings, ")
	fmt.Print("human")
	fmt.Print("\nHow is life on Earth?")
}

Println VS. Print - Example Output

Variables

  • You can declare a variable type by using the "var" keyword followed by the variable name and finally the variable type
  • For example:
    • 
var x int = 5
  • Alternativly, you can implicitly declare a variable and have the interpreter or compiler figure out the type itself
  • For example:
    • 
x := 5

Variables - Example


package main

import "fmt"

func main() {
	var x int = 5 //integer variable!
	
	y := 10 //Implicit declaration for integer variable
	
	var a, b, c string = "a", "b", "c" //Declaration of a list of strings
	
	i, j := "string", true //Implicit declaration of a string and a boolean
	
	//Print all the variables
	fmt.Println("x:", x, "\ny:", y, "\na:", a, "\nb:", b, "\nc:", c, "\ni:", i, "\nj:", j)
}

IF Statements

  • A control statement in which the result determines whether preceding statements will run or not
  • In Go brackets () are not required around the control statement
  • However, the braces {} surrounding the remaining statements are required

IF Statements - Example

  • Try out these basic IF statements
    • 
package main

import "fmt"

func main() {
	x := 10
	
	if x < 100 {
		fmt.Println(x, "is less than 100")
	}
	
	if !false {
		fmt.Println("The opposite of false is true!")
	}
	
	if true && x > 50 {
		fmt.Println("Boolean AND!")
	}
	
	if true || x > 50 {
		fmt.Println("Boolean OR!")
	}
}
  • Which of these statements will print and which ones will not?

IF Statements - Example Output

The opposite of false is true!
Boolean OR!

IF-ELSE Statements

  • When the condition of the IF statement is false, you can choose to have a different statement(s) run
  • These statements are surrounded by the ELSE statement

IF-ELSE Statements - Example

  • Placement of the ELSE statement matters!
    • 
package main

import "fmt"

func main() {
	
	x := 10
		
	if x < 5 {
		fmt.Println(x, "< 5")
	}
	else {
		fmt.Println(x, ">=5")
	}
}

FOR Loops

  • A loop iterates over a statement(s) a set number of times
  • A FOR loop needs to be given a start, a condition and a 'step counter'

FOR Loops - Example

  • We are going add the numbers from 0-9
    • 
package main

import "fmt"

func main() {
	sum := 0
	for i := 0; i < 10; i++ {
		sum += i
	}
	fmt.Println(sum)
}
  • Why only until 9, when the condition says less than 10?

FOR Loops - Example Output

45

Program exited.

'WHILE' Loops

  • WHILE loops run until a condition is met to stop the loop
  • No start and no 'step counter'
  • Every FOR loop can be rewritten as a WHILE

'WHILE' Loops - Go Style

  • There is no actual 'WHILE' loop in Go, it's just a FOR without the start and 'step counter'
  • Let's print 'Hello World!' while i is less than 5
    • 
package main

import "fmt"

func main() {
	i := 0
	for i < 5 {
		fmt.Println("Hello World!")
		i++
	}
}

Functions

  • A named section of a program that performs a specific task
  • You can write functions yourself or they can be imported in packages
  • Examples of functions that are imported in packages are:
    • Println - from the "fmt" package
    • sqrt - from the "math" package

Functions - Writing your own

  • Funtion definitions start with the "func" keyword
  • The argument types need to be declared as well as the return type, if applicable

Functions - Example

  • Writing a function to add 2 integers
    • 
package main

import "fmt"

func add(x int, y int) int {
	return x + y
}

func main() {
	fmt.Println(add(42, 13))
}
